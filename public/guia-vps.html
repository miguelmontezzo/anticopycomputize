03

Criar um usuário dedicado

Segurança e boas práticas

Rodar tudo como root não é recomendado. Cria um usuário dedicado — isso também evita qualquer problema de permissão com o Claude Code:

Terminal
$ adduser claudeuser
# Cria o novo usuário — preencha a senha, o resto pode ficar em branco

Adiciona o usuário ao grupo sudo:

Terminal
$ usermod -aG sudo claudeuser

Troca para esse usuário:

Terminal
$ su - claudeuser
// Importante
A partir daqui todos os comandos rodam como claudeuser, não como root.
04

Atualizar o sistema e instalar o Node.js

Preparando o ambiente

Atualiza os pacotes do sistema:

Terminal
$ sudo apt update && sudo apt upgrade -y
# Atualiza todos os pacotes do sistema

Instala o Node.js 20:

Terminal
$ curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
$ sudo apt install -y nodejs

Verifica se instalou certo:

Terminal
$ node --version
v20.x.x
// Verificação
Deve aparecer algo como v20.x.x. Se aparecer, tudo certo — próximo passo!

06

Manter o Claude Code rodando em background

Ativo 24/7 com PM2

Quando você fechar o terminal, o processo para. O PM2 resolve isso mantendo tudo ativo 24/7:

Terminal
$ npm install -g pm2

Cria uma pasta de trabalho e sobe o processo:

Terminal
$ mkdir ~/meu-projeto && cd ~/meu-projeto
$ pm2 start "claude" --name claude-code

Garante que o PM2 reinicia automaticamente se a VPS for reiniciada:

Terminal
$ pm2 startup
$ pm2 save
// Resultado
Agora o Claude Code fica ativo 24/7 mesmo com o terminal fechado.

Computador liberado, Claude rodando no servidor

Sempre que quiser retomar, abre o terminal e conecta:

// Mac / Linux — Terminal

Terminal
